# Contributing

Contributions are welcome after the first public release.

## Guidelines

- Keep `SKILL.md` concise and operational.
- Put detailed style, prompt, and rubric content in `references/`.
- Add examples when adding a new style or workflow.
- Verify asset rights before adding images, fonts, icons, or templates.
- Prefer short, readable image text in examples.
- Test Vietnamese accents when adding Vietnamese examples.

## Pull Request Checklist

- The skill still has valid YAML frontmatter.
- New styles include selection rules and prompt guidance.
- New assets include license notes.
- Examples do not contain private data.
- Documentation matches the installable skill folder.
